# JobTrail

A zero-dependency, open-source **job application tracker**. Paste an Indeed or
LinkedIn link, get the role parsed into clean structured data, and track the
whole application — status, dates, resume, cover letter, notes — in a local web
UI. Comes with a matching CLI for batch markdown export.

- **Zero dependencies.** Pure Python standard library (3.9+). No `pip install`,
  no `requests`, no framework, no headless browser. The web server is
  `http.server`, storage is `sqlite3`, the frontend is plain HTML/CSS/JS.
- **Local & private.** Runs on `127.0.0.1` only. Your job data and uploaded
  files never leave your machine (`data/` is git-ignored).
- **Indeed + LinkedIn.** Both embed schema.org `JobPosting` data; LinkedIn is
  read through its public no-auth guest endpoint.

## Two ways to use it

### 1. The tracker UI (recommended)

```bash
cd jobtrail
./bin/job-tracker            # serves http://127.0.0.1:8765
# open that URL in your browser
```

In the UI you can:

- **Add a job** by pasting an Indeed or LinkedIn URL — it fetches and parses
  automatically.
- **Track each application**: status pipeline (Saved → Applied → Interviewing →
  Offer → Rejected / Archived), date applied, follow-up date, salary
  expectation, recruiter contact, a 1–5 interest rating, and free-form notes.
- **Attach files**: upload a resume and cover letter per job; download or remove
  them anytime.
- **Read the full description** rendered as Markdown, or export it via the
  Markdown button.

All edits autosave to the local SQLite database.

**Prefer an app icon?** Run `python3 install-desktop.py` once to add a clickable
JobTrail icon to your desktop / app menu (Linux, macOS, Windows). Launching it
starts the server and opens the UI for you — see
[docs/DESKTOP_APP.md](docs/DESKTOP_APP.md). Your data and API keys stay in the
git-ignored `data/` directory, so nothing private is ever published.

### 2. The CLI (batch markdown export)

```bash
# one job to the terminal
./bin/indeed-cli "https://www.indeed.com/viewjob?jk=JOBKEY" --stdout

# a batch -> Markdown files in ./output
./bin/indeed-cli URL1 URL2 https://www.linkedin.com/jobs/view/123456 --file more.txt

# structured JSON, or parse a saved page offline
./bin/indeed-cli "<url>" --json
./bin/indeed-cli --html saved.html --url "<original url>" --stdout
```

The CLI handles Indeed and LinkedIn too (`--help` for all flags).

## Your data is safe by default

Jobs and their tracking categories are never lost by accident:

- **Delete is reversible.** Deleting a job moves it to **Trash** (its files are
  kept). An *Undo* appears immediately, and you can **Restore** from the Trash
  view anytime. Only **Delete forever** removes it permanently — and even that
  takes a backup first.
- **Automatic backups.** A consistent snapshot of the database is taken on
  startup, on shutdown, before every destructive action, and periodically
  whenever data has changed. Snapshots use SQLite's online-backup API and are
  pruned to a rolling window of 60. They live in `data/backups/`.
- **Off-machine copies.** Click **Export backup** for a single JSON file of
  everything (re-import with **Import** — it merges and never overwrites
  existing jobs). Point automatic backups at a cloud-synced folder with
  `./bin/job-tracker --backup-dir "/path/to/Dropbox/JobTrail"`.
- **Safe upgrades.** New columns are added by idempotent migrations, so your
  existing database keeps all its data across updates.

Restoring a snapshot is just copying it back:
`cp data/backups/jobs-<timestamp>.db data/jobs.db` (while the server is stopped).

## Why from scratch (not CLI Printing Press)

This workspace's `lichess-cli` was generated by **CLI Printing Press** from
Lichess's official **OpenAPI spec**. Neither Indeed nor LinkedIn publishes an
equivalent public spec for "job URL → details" (their job APIs are partner-gated
or deprecated), so the spec-to-CLI generator has nothing to consume. Both sites
*do* ship a machine-readable `JobPosting` object inside each page, so JobTrail
targets that directly.

## Anti-bot note

Indeed and LinkedIn challenge automated traffic. JobTrail sends realistic
browser headers and retries with backoff, but a direct fetch can still return
**HTTP 403** from some networks. Fallbacks:

- **CLI:** open the job in your browser, *Save Page As… → HTML*, then
  `./bin/indeed-cli --html saved.html --url "<original url>"` — identical output.
- Keep request volume modest and respect each site's Terms of Service.

## Layout

```
jobtrail/
  bin/
    job-tracker          # launch the web UI
    indeed-cli           # CLI for markdown/JSON export
  indeed_cli/            # parsing + rendering engine (shared)
    sources.py           # route a URL -> indeed / linkedin / generic
    fetch.py             # urllib fetch: browser headers, gzip, retry/backoff
    parse.py             # JSON-LD / _initialData -> normalized Job
    linkedin.py          # LinkedIn guest-endpoint source
    htmlmd.py            # HTML description -> Markdown (stdlib parser)
    render.py            # Job -> Markdown document
  webapp/
    server.py            # stdlib HTTP server + JSON API (127.0.0.1)
    db.py                # sqlite3 persistence (soft delete, trash, export/import)
    backup.py            # online snapshots, rolling retention, periodic backups
    static/              # index.html, styles.css, app.js (no framework)
  examples/              # sample Indeed + LinkedIn pages, urls.txt
  tests/                 # offline regression + loopback server tests
  data/                  # sqlite db + uploaded files (git-ignored)
  output/                # CLI markdown exports (git-ignored)
```
